Architecture

Phantom is a Rust workspace with 36 crates grouped into nine operational domains. Each crate is either OPERATIONAL (✅), SKELETAL (🔧) or PHASE 2+ (🚧). The Core, Rendering, Terminal, AI, and Federation rings form the substrate. The Capture, Voice, History, and Extensibility rings extend it.

Domain · Core system

The boot kernel. Without these, nothing else loads.

phantom✅
Main binary. winit event loop, GPU init, panic recovery, supervisor socket handshake. crates/phantom/src/main.rs
phantom-supervisor✅
Erlang/OTP-style monitor. Heartbeat watch + auto-restart with back-off. Issue #53.
phantom-app✅
Application lifecycle. Pane management, boot sequence, agent coordination, inspector UI. Issues #97, #44.
phantom-protocol✅
Unix-socket IPC wire protocol between supervisor and app. Heartbeat, events, commands.

Domain · Rendering

phantom-renderer✅
wgpu GPU pipeline. Text atlas, quad batcher, CRT post-fx, screenshot, video. Issues #82–#84.
phantom-ui✅

phantom-scene✅
Retained scene graph with dirty-bit tracking, clock/cadence, frame delta clamp.

Workspace topology

Visualized as a dependency ring. Inner rings are loaded first at boot.

                          ┌─────────────────────────────┐
                          │      phantom (binary)       │
                          └──────────────┬──────────────┘
                                         │
              ┌──────────────────────────┼──────────────────────────┐
              │                          │                          │
       phantom-supervisor        phantom-protocol           phantom-app
        (sibling proc)        (IPC, events, heartbeat)   (panes, boot, ui)
                                         │
              ┌──────────────────────────┼──────────────────────────┐
              │             │            │            │             │
      phantom-renderer  phantom-ui  phantom-scene  phantom-terminal phantom-context
       (wgpu/postfx)   (tokens)    (graph)         (pty/vt100)      (project)
                                         │
              ┌──────────────────────────┼──────────────────────────┐
              │             │            │            │             │
       phantom-agents  phantom-brain phantom-loop phantom-nlp  phantom-semantic
       (12 roles)      (ooda/ledger) (overseer)   (interpret)  (parser)
                                         │
              ┌──────────────────────────┼──────────────────────────┐
              │             │            │            │             │
      phantom-builder phantom-fleet phantom-adapter  phantom-mcp   phantom-plugins
       (1 repo)      (N apps)      (app abstraction) (mcp)         (wasm)
Pattern
Every "outer ring" crate is replaceable behind a trait. The inner rings depend only on the traits, never the concrete crates. This is what lets phantom loop run drive the substrate without booting wgpu/winit.
