SECTION 08 · LOOP OVERSEER SYS://loop.0

Loop Overseer

The loop overseer is what lets Phantom run autonomous work pipelines without a human in the chair. A LoopRunner is a 6-state async FSM that pulls typed input from a LoopSource, dispatches it through the substrate (which spawns an agent), waits for a schema-valid complete_task, validates the payload, and fires any on_complete effects. Repeat forever.

The LoopSource trait

Every loop is defined by what it pulls. The LoopSource trait is the pluggable input boundary: 

pub trait LoopSource {
    fn next(&mut self) -> LoopPullResult;
}

pub enum LoopPullResult {
    Available(LoopInput),
    Empty,
    Done,
    Error(SourceError),
}
CronSourcepoll
Agentless poll loops on a fixed schedule. Used for periodic substrate maintenance.
LoopMessageQueueSourcequeue
Drains LoopQueueRegistry by queue name. Used for cross-loop fan-out (implementer → review).
GhIssueQueueSourcegithub
gh issue list adapter. Yields one LoopInput per matching issue.
GhPrReviewQueueSourcegithub
gh pr list adapter for open PRs awaiting review.

The substrate dispatcher

SubstrateAgentDispatcher at phantom-loop/src/dispatcher/substrate.rs:98 is the production AgentDispatcher trait impl. It pushes a SpawnSubagentRequest onto a shared queue, allocates an AgentId from a monotonic counter (base SUBSTRATE_DISPATCHER_ID_BASE = 50_000 so substrate IDs don't collide with UI-spawned agents), and registers a pending oneshot keyed by that ID. 

impl AgentDispatcher for SubstrateAgentDispatcher {
    fn dispatch(&self, input: LoopInput) -> DispatchHandle {
        let agent_id = self.next_id();
        let (tx, rx) = oneshot::channel();
        self.pending.lock().insert(agent_id, tx);
        self.queue.push(SpawnSubagentRequest {
            agent_id,
            role:     input.role(),
            task:     input.task(),
            opts:     AgentSpawnOpts::new(input.role(), input.task())
                          .with_requires_complete_task(true)
                          .with_exit_schema(input.exit_schema()),
        });
        DispatchHandle { agent_id, rx }
    }
}

SubstrateBackend — pluggable execution

The dispatcher only pushes. The SubstrateDriver pops. The driver delegates each request to a SubstrateBackend impl. Two impls ship: production (Claude API) and mock (tests). 

pub trait SubstrateBackend {
    async fn drive_agent(
        &self,
        request: SpawnSubagentRequest,
    ) -> Result<serde_json::Value, DispatchError>;
}

The four shipped loop specs

Live at .phantom/loops/*.toml. Each spec is a self-describing TOML record consumed by LoopRunner. The implementer spec, for example: 

# .phantom/loops/implementer.toml
id = "implementer"
max_concurrent = 1
on_quarantine = "skip_and_continue"

[agent]
role        = "Actor"
allow_tools = ["read_file", "write_file", "edit_file", "run_command", "gh_pr_create"]
system_prompt = """
You are an implementation agent for the Phantom repo (jdmiranda/phantom).
Pull the assigned issue. Read the relevant code. Write tests first.
Implement the change. Run cargo test. Push a branch. Open a PR.
You MUST call complete_task with {issue_number, pr_url, summary}.
"""

[agent.policy]
auto_approve    = true
timeout_seconds = 1800

[agent.exit_schema]
type     = "object"
required = ["issue_number", "pr_url", "summary"]

[source]
kind = "queue"
name = "implementer-queue"

[[on_complete]]
kind  = "enqueue_to"
queue = "review-queue"

[[on_complete.fields]]
from = "result.pr_url"
to   = "pr_url"
pr_finder_review.tomlSCAN
Polls GitHub for open PRs. Yields one input per PR awaiting review.
pr_finder_impl.tomlSCAN
Polls GitHub for open issues. Yields one input per labeled, unassigned issue.
implementer.tomlWRITE
Drains implementer-queue. Writes code, runs cargo test, opens PR.
reviewer.tomlREAD
Drains review-queue. Reads PR diff, posts review comments.

Running it

phantom loop run --repo ~/code/phantom --loops pr_finder_impl,implementer,pr_finder_review,reviewer

Preflight gates

Before any runner starts, phantom-loop::preflight runs three checks:

check_gh_binaryG1
gh on PATH and executable. No fallback to git + REST — the loop refuses to start without gh.
check_gh_authG2
gh auth status reports a logged-in account. Otherwise refuse.
check_mcp_collisionsG3
MCP tool names cannot collide with reserved complete_task / abort_task lifecycle names.

The RunLock

Two concurrent loop runners against the same repo would race. Phantom prevents this with an exclusive file lock at <repo>/.phantom/loops/.runlock, acquired by RunLock::acquire(&repo) on startup and released via Drop on Ctrl-C or process exit.

The headless substrate driver

phantom loop run doesn't boot wgpu, winit, or the renderer. The SubstrateDriver (PR #670) is a pure tokio async tick loop. It drains the SpawnSubagentQueue, drives each request through ChatBackedSubstrateBackend, and emits Event::AgentTaskComplete onto a tokio mpsc bus. A SubstrateCompletionRouter subscribes to that bus and fulfils the pending oneshot. Same dispatcher, same agent contract, no GUI.
