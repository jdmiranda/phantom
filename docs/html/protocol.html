Event Protocol

Everything that happens in Phantom is an Event. The bus is typed — 22 variants, 8 topic categories — and routed through EventTopic. Subscribers filter by topic, not by ad-hoc string matching. The supervisor/app IPC reuses the same envelope.

EventTopic routing

Every Event maps to exactly one EventTopic. Subscribers ask for a topic set, not a regex. This makes routing cheap and the dispatch table compile-time complete. 

pub enum EventTopic {
    Terminal,    // TerminalOutput · CommandStarted · CommandComplete · …
    Agents,      // AgentSpawned · AgentProgress · AgentTaskComplete · …
    Sessions,    // SessionSwitched
    Focus,       // FocusChanged
    Brain,       // BrainDecision · NlpInterpreted
    Video,       // VideoPlaybackStateChanged · FrameCaptured
    Fx,          // GlitchFxTriggered
    System,      // MemoryPressure · JobCompleted · Shutdown
    Custom,      // Custom escape hatch
}

Wire format

Length-prefixed JSON over Unix domain socket. The supervisor/app messages wrap an inner Event: 

#[derive(Serialize, Deserialize)]
#[serde(tag = "type")]
pub enum AppMessage {
    Heartbeat,                      // liveness ping
    Event     { event: Event },        // forward an event for monitoring
    Shutdown,                       // clean exit; do not respawn
}

pub enum SupervisorMessage {
    Restart,                        // graceful restart (config reload etc.)
    Quit,                           // final shutdown request
}

FastPathKind

pub enum FastPathKind {
    AutoApprove,    // AwaitingApproval skipped because auto_approve=true
    SkipPlanning,   // Planning skipped because the task was simple enough
}

Every emission carries a human-readable reason. The audit log is grep-able for regressions — if a gate skip ever happens for the wrong reason, it surfaces here, not silently in code.

LoopMessage (cross-loop)

Separate but related: typed messages between loops live in phantom-loop::LoopMessage. They use the same tagged-enum + JSON pattern, routed through LoopQueueRegistry rather than the main event bus. 

pub struct LoopMessage {
    pub from_loop:    String,
    pub to_queue:     String,
    pub message_kind: String,             // schema discriminator
    pub payload:      serde_json::Value,    // validated against the queue's schema
    pub trace_id:     String,
    pub timestamp:    SystemTime,
}
Why a typed event bus matters
Untyped event buses degrade into RPCs disguised as broadcasts. With Event as a tagged union, every consumer must handle every variant — or explicitly opt out via _. New variants force compile errors at every subscriber. The match is the contract.
