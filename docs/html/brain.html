SECTION 06 · BRAIN · AMBIENT OODA SYS://brain.0

Brain · OODA

The brain runs an ambient OODA loop on a dedicated thread. It Observes the event stream, Orients facts onto a confidence scale, Decides through the TaskLedger guarded mutator, and Acts by emitting typed AiActions onto a channel. Plans aren't strings — they're a vector of typed PlanStep records with a 9-state status FSM and typed failure causes.

The central data structure. Holds a goal, a vector of facts, a vector of typed plan steps, and a history of superseded plans. Defined at phantom-brain/src/orchestrator.rs:~230.

DispatchBlocked — typed refusals

No string excuses. Every refusal is a typed variant the caller must handle. 

pub enum DispatchBlocked {
    OutOfBounds   { idx: usize, len: usize },
    NotPending    { idx: usize, current: PlanStepStatus },
    UnmetDeps     { idx: usize, open: Vec<usize> },
    Quarantined,
}

StepFailureCause — typed cascade semantics

When a plan step fails, the brain doesn't just shrug. The cause is typed, and the cause drives the QuarantinePolicy. 

pub enum StepFailureCause {
    AgentFailed,
    AgentQuarantined     { agent_id: AgentId, since_ms: u64 },
    DispatchTimedOut,
    DependencyFailed     { dep_idx: usize },
    CompleteTaskNotCalled,
}

pub enum QuarantinePolicy {
    FailAndAllowRetry,   // default: mark failed, clear agent_id, fresh spawn ok
    FailAndCascade,      // + mark downstream depends_on steps as Skipped
    Park,                 // leave Active, log cause; human must intervene
}

The action channel

The brain emits AiAction records onto a tokio mpsc. The forwarder thread translates each variant into a side-effect: spawning an agent, enqueuing a loop message, emitting a UI event. 

pub enum AiAction {
    SpawnAgent           { role: AgentRole, task: String, opts: AgentSpawnOpts },
    EnqueueLoopMessage   { queue: String, message: LoopMessage },

    RejectStep           { ledger: LedgerId, step: usize, reason: String },

}

FactConfidence — typed knowledge

Every fact the brain consumes is labeled. A Verified fact came from a tool result; a Guess came from the model. Plan steps that depend on guesses get extra scrutiny. 

pub enum FactConfidence {
    Verified,    // produced by a tool with structured output
    ToLookUp,    // brain knows it doesn't know; queued for retrieval
    ToDerive,    // computable from existing facts
    Guess,       // model assertion, unverified
}

Stall detection

Every tick the ledger updates a stall_counter. If steps stop transitioning and no new facts arrive, the counter increments. When it crosses stall_threshold, the brain pushes the current plan onto plan_history, clears plan, and re-enters the outer loop. New plan, possibly new strategy, same goal.

Why a separate process from the brain isn't enough
The brain is not "the AI that talks to the user." That's Conversational, an agent. The brain is the supervisory cortex — it never produces a chat response. Its only outputs are typed actions. This keeps the user-facing surface separable from the planning substrate.
