SECTION 02 ¬∑ CINEMATIC BOOT SYS://boot.0

Boot Sequence

Phantom doesn't open. It boots. A 7-phase state machine drives a 6-second cinematic warm-up: static noise, CRT warm-up, glitch-typed logo reveal, animated system check, welcome handoff. The bezel flickers, the scan beam sweeps, the phosphor settles, and only then does the terminal accept your input.

Phases
7
Total duration
6.5 sec
Driver
phantom-app/src/boot.rs
Skippable
--no-boot

State machine

The BootSequence state at boot.rs:1 is a tagged enum advanced by frame delta. Each variant carries its own progress timer (in seconds).

pub enum BootSequence {
    BlackScreen      { t: f32 },     // 0.0 ‚ÄĒ 0.5s  static noise
    CrtWarmup        { t: f32 },     // 0.5 ‚ÄĒ 1.5s  centre-out clear, scan beam
    LogoReveal       { t: f32 },     // 1.5 ‚ÄĒ 3.5s  glitch-typed PHANTOM
    SystemCheck      { t: f32 },     // 3.5 ‚ÄĒ 6.0s  4 status lines w/ bars
    Welcome          { t: f32 },     // 6.0+        cursor blink, waits key
    TransitionToTerm { t: f32 },     // 0.5s        fade to terminal
    Done,
}

Timeline

The noise clears center-out, sweeping outward as a phosphor scan beam crosses from top to bottom. This is the tube coming alive. The post-fx scanline_intensity uniform ramps from 0.0 to its theme default over the 1.0s of this phase.
[ T = 1.5s ] ¬∑ phase 3

The logo

Defined as a static &[&str] at boot.rs:85-93. Unicode box-drawing, 9 rows tall. During reveal, each glyph position is replaced with a randomly-chosen glitch char that settles after a per-cell timer expires.

Skipping boot

For development iteration:

# skip the cinematic, go straight to the prompt
cargo run -- --no-boot

# or, in code, App is constructed with boot=Done
App::with_config_scaled(gpu, config, sock, scale).skip_boot();
Aesthetic
Phantom is a diegetic computing surface: a piece of in-world hardware the operator boots up ‚ÄĒ not an app that opens. Phosphor type on a CRT, a deliberate cold-start, and a power-on self-test that means something. The boot sequence is part of the product, not a splash screen.

Why this matters

The boot sequence is not decoration. It exercises the renderer's hot path before the user's first command, warms the glyph atlas (every box-drawing char it'll ever use lands during the logo reveal), and lets the supervisor confirm the heartbeat handshake before any real work flows through. By the time you see SYSTEM READY, the GPU, the PTY, the agent mesh, and the supervisor socket are all hot.
