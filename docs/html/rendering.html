Rendering Stack

Phantom is GPU-first. Every pixel is composed by wgpu on whichever backend the host provides — Metal on macOS, Vulkan on Linux, D3D12 on Windows. A two-stage pipeline renders the scene to an offscreen texture, then composites it through a CRT post-fx pass with six physically-inspired effects.

Core types

The renderer is organized around GpuContext as the single owner of device, queue, surface, and surface configuration. Every renderer is constructed once and recycles per-frame state.

The CRT post-fx pass

The composite shader is shaders/crt.wgsl. Six effects are evaluated in order. Each effect samples the offscreen texture (warped UV from the previous step). The full-screen triangle is the only geometry the post-fx pass draws. 

// crt.wgsl :: composite order
fn fs_main(in: VertexOut) -> vec4<f32> {
    let uv         = barrel_distort(in.uv, params.curvature);
    if (out_of_bezel(uv)) { return vec4<f32>(0.0); }
    let aberrated  = chromatic_aberration(uv);
    let bloomed    = phosphor_bloom(aberrated, params.glow_color);
    let scanned   = scanlines(bloomed, in.pos.y);
    let vignetted = vignette(scanned, uv);
    let grained   = film_grain(vignetted, params.time);
    return grained;
}
Barrel distortioncrt.wgsl:97-141
Warps UVs outward from center by 1.0 + strength · r², simulating CRT tube curvature. Pixels outside the bezel return pure black.
Chromatic aberrationcrt.wgsl:152-163
Splits R/G/B channels radially. Offset scales with distance from center, capped at 0.02 · aberration.
Phosphor bloomcrt.wgsl:172-190
8-tap box blur tinted by glow_color, blended additively at 0.7 · bloom_intensity. The signature look.
Scanlinescrt.wgsl:199-204
Sinusoidal darkening: 1.0 − intensity · 0.3 · (1 + sin(y · 2π)) · 0.5. Per-pixel-row modulation.
Vignettecrt.wgsl:213-219
smoothstep(0.2, 0.7071) radial fade. Multiplies edges by 0.85.
Film graincrt.wgsl:228-233
Animated hash-based noise. Temporal variation via the time uniform; grain = (hash − 0.5) · 0.12 · intensity.

Six themes ship in phantom-ui

Each theme is a Theme struct combining TerminalColors (16-color ANSI palette + cursor + selection), ShaderParams (the 10 CRT uniforms above), and UiColors (chrome). The CRT effect strengths are tuned per theme — Phosphor is restrained, Amber is aggressive, Vapor leans into chromatic split.

Run with any theme:

cargo run -- --theme amber
cargo run -- --theme vapor
cargo run -- --theme verdant
cargo run -- --theme plain   # disables all CRT effects

Hot shader reload

During development, recompile and re-bind a shader without restarting: 

PHANTOM_HOT_SHADERS=1 cargo run --bin phantom-supervisor

The PostFx pipeline polls shaders/crt.wgsl on mtime change and calls reload_shader() at postfx.rs:51-68. The pipeline rebuilds, the next frame uses the new shader. Useful for tuning the bloom radius or grain noise live.

Mixed content

Every Phantom pane composes terminal text + rich widgets + inline images + shader FX in the same pass. The renderer doesn't distinguish "terminal mode" from "widget mode" — it draws a sequence of primitives (Quad, Glyph, ColorGlyph, VideoFrame) into the offscreen buffer, then the post-fx pass tonemaps and CRT-warps the whole thing. An agent's progress bar lives next to your ls output and they both phosphor-bloom together.

Window occlusion
When the window is minimized or hidden, WindowEvent::Occluded suppresses the render loop entirely (phantom/src/main.rs:186-197). Idle Phantom does not burn GPU.
