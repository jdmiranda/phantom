The Manifesto

Every terminal emulator in 2026 is a dumb glass window. WezTerm is beautiful glass. Ghostty is faster glass. Kitty is glass with pictures. Alacritty is the fastest glass that ever glassed. Forty-eight years of innovation gave us… faster font rendering.

The AI terminals didn't fix this. Warp bolted a chatbot onto a terminal. Wave tried inline widgets and got abandoned. Fig sold to AWS and died. None of them asked the question: what if the terminal wasn't a window you type into, but a living system that thinks alongside you?

The Four Properties

All four are landed on main. The 5 audit gaps tracked under issue #650 are closed.

1 · Per-role tool whitelistsLANDED
Every tool-use routes through capability::check_capability(ctx.role, tool.class()) in phantom-agents/src/dispatch/mod.rs. Denials return a canonical "capability denied: <Class> not in <Role> manifest" so the model self-corrects. PR #655 made the event log non-optional at the gate so it cannot be silently bypassed.
2 · External state machineLANDED
phantom-brain::TaskLedger with try_dispatch(idx) → Result<&PlanStep, DispatchBlocked> and the 9-state AgentStatus FSM. StepFailureCause + QuarantinePolicy give typed cascade semantics on each PlanStep.
3 · Structured exitLANDED
The complete_task lifecycle tool fires only when spawned with AgentSpawnOpts::with_requires_complete_task(true). Payload validated against the per-loop ExitSchema. 3 consecutive invalid calls flatline the pane. The legacy stringly-typed PARTIAL exit is gone.
4 · Typed inter-agent messagingLANDED
phantom-protocol::Event bus with 22 typed variants and EventTopic routing. FastPathTaken { agent_id, kind, reason } audits every gate skip. phantom-loop::LoopMessage handles cross-loop fan-out through LoopQueueRegistry.

The closed loop

The four harness properties sum to a closed feedback loop. The brain auto-discovers work in its own repository, scores it for utility, and forwards the highest-scoring candidates to the loop overseer for autonomous implementation. Phantom-on-Phantom. 

 phantom-brain GoalSource polls jdmiranda/phantom
     -> score_candidate weighted-sum scorer
     -> HardExclusions + TrustBand + RateLimiter window
     -> AiAction::EnqueueLoopMessage
     -> LoopQueueRegistry.push(implementer-queue, LoopMessage{...})
     -> LoopRunner pulls via LoopMessageQueueSource
     -> SubstrateAgentDispatcher builds AgentSpawnOpts with requires_complete_task=true
     -> SubstrateDriver.tick drains SpawnSubagentQueue
     -> ChatBackedSubstrateBackend streams TextDelta / ToolUse to Claude
     -> agent calls complete_task with typed payload
     -> Event::AgentTaskComplete fulfils dispatcher oneshot
     -> LoopRunner validates payload against ExitSchema
     -> on_complete LoopEffects fire (typically EnqueueTo review-queue)
