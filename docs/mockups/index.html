PHANTOM · UX RESET · 2026-05-20

A canvas of apps, one design system.

Phantom isn't a chat window — it's a container for spatial citizens. Every app (Terminal, Agent, Inspector, Monitor, Video, …) implements AppAdapter, declares a SpatialPreference, and renders inside the rect the arbiter grants it using the shared Tokens. Themes swap the tokens; every surface redraws live. Pub/sub on the event bus wires apps to each other.

01
Every app is a spatial citizen
Each adapter declares SpatialPreference { min, preferred, max, priority }. The arbiter resolves competing claims; the adapter renders inside whatever rect it gets.
02
Apps compose via pub/sub
Terminal publishes terminal.output; Agent subscribes. Agent publishes agent.event; Inspector subscribes. Like Unix pipes for structured data between GUI apps.
03
Themes are token swaps
One Tokens shape; six palettes. Phosphor, amber, ice, blood, vapor (Miami Vice), cyber (cyberpunk). Aesthetic style, not franchise IP.

What's NOT in these

No Fallout (no RobCo, no MU/TH/UR, no Vault-Tec, no Pip-Boy). No Alien IP (no Weyland-Yutani). The aesthetic styles (phosphor CRT, neon Miami Vice, cyberpunk magenta) stay — just stripped of franchise marks. CRT scanlines are an opt-in shader, not a default.

Mockups in docs/mockups/. Shared stylesheet: system.css — single source of truth for tokens and components. The Rust side mirrors the same shape in phantom_ui::tokens + phantom_ui::themes.
